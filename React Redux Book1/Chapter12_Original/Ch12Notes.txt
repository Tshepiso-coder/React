Class vs Functional Components
Both class-based and functional-based components serve the same purpose in React — they 
produce HTML to the user via JSX and handle user feedback. 
The key difference lies in how they do it and how clean the code is.

Class-based components have been around since the beginning of React and handle 
complex logic like state management and lifecycle methods out of the box. 
However they require more boilerplate code and the this keyword can be confusing for 
beginners.

Functional components on the other hand are simpler and cleaner to write. 
With the introduction of Hooks (useState, useEffect etc.), functional components can 
now do everything class components can — managing state, handling lifecycle events, 
network requests, timers and more. Hooks also make it significantly easier to share 
logic between components, something that was messy and complicated to do with class 
components.

The modern standard in React development is to use functional components with Hooks as 
the code is cleaner, easier to read and requires less boilerplate.
However, as future developers you will encounter both in the real world — older 
codebases will likely use class components while newer projects will use 
functional components with Hooks. It is therefore important to be comfortable reading 
and understanding both.


EXAMPLE: Sharing Logic between Components
1) Class based components
// You have to create a WRAPPER class just to share the logic
class MouseTracker extends Component {
  state = { x: 0, y: 0 }
  
  handleMouseMove = (e) => {
    this.setState({ x: e.clientX, y: e.clientY });
  }

  render() {
    // Have to wrap every component that needs this logic
    return this.props.render(this.state);
  }
}

// Then in every component that needs it, you wrap it like this
<MouseTracker render={({ x, y }) => (
  <Cart x={x} y={y} />
)} />


2) Function based components
// Write the logic ONCE in a custom hook
function useMousePosition() {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  
  useEffect(() => {
    window.addEventListener('mousemove', (e) => {
      setPosition({ x: e.clientX, y: e.clientY });
    });
  }, []);

  return position;
}

// Then use it in ANY component with just one line!
function Cart() {
  const { x, y } = useMousePosition(); // done!
}

function AddProduct() {
  const { x, y } = useMousePosition(); // same logic, reused cleanly!
}
